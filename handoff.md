# Handoff — PolyDock Inventory & Design Tool
_Last updated: 2026-07-06 · Current stage: Stage 3 — Design Canvas (done, extended) · Stage 4 up next_

## 🎯 Goals
Give staff a 2D canvas to lay out real PolyDock modules at true scale, snapping edge-to-edge with correct seam classification (straight vs. notched connector needed), plus the accessories/gangways/background-photo tooling needed to make a design actually match a real dock job. Done and verified; Stage 4 (stock-aware BOM) is the next real milestone — it's what actually consumes the seam data this stage produces.

## 📍 Current State
Stages 1–3 are all functionally complete and verified in-browser. The app has: a tabbed inventory UI with CSV bulk import (Stages 1–2), and a full design canvas (Stage 3) — a "Designs" tab where staff create/list/edit designs, drag real dock modules onto an SVG grid at true scale, snap them flush, rotate in 90° steps, and see every seam automatically classified as needing a straight or notched connector (color-coded). This session added four user-requested features on top of that: (1) a module stays selected and rotatable after a drag, not just on first placement; (2) an Accessories palette tab for freely-placed items (cleats, bumpers, ladders, hinges, etc. — no footprint, no snapping); (3) a Gangways palette tab for true-scale, snappable gangway/ramp modules whose attachment seam to a dock module renders in a distinct dashed-purple style (not a `CONNECTOR_RULES` straight/notched seam — gangways attach differently); (4) a background photo (e.g. a Google Maps screenshot of the lakefront) that staff can upload, compress, and freely position/scale/fade behind the grid to align a design to the real property. Designs save/reload correctly, including the background image. Still no backend — everything is client-side localStorage.

## 📂 Files I'm Working On
- `src/features/design/geometry.js` — the core geometry engine: footprint/bounds from rotation, snap-to-adjacent-edge, seam/junction classification (3+-way corner convergence), and now `hasFootprint(part)` — the single source of truth distinguishing snappable footprint items (modules/gangways) from freely-placed markers (accessories) — plus `isGangwayAttachment` seam gating.
- `src/features/design/DesignCanvas.jsx` — the SVG canvas: pointer-based drag/snap with SVG-root-level move/up handlers, click-vs-drag distinction, memoized footprint/marker item splits, a mutual-exclusion guard between module-drag and background-drag state, and the background `<image>` layer (pointer-events: none so it never blocks deselect-clicks).
- `src/features/design/DesignEditor.jsx` — toolbar (name, free-design toggle, rotate/delete/save) plus background upload/position/scale/opacity controls; dirty-check now covers the background image too and is reported up via `onDirtyChange` for the top-level nav guard.
- `src/features/design/Palette.jsx` — replaces the old `ModulePalette.jsx`; now tabs across Modules/Gangways/Accessories, reusing the shared `CategoryTabs` component instead of duplicating tab markup.
- `src/features/design/AccessoryMarker.jsx`, `src/features/design/imageUtils.js` — new: the marker-shape renderer for footprint-less items, and the upload-time image compression helper (max 1600px, JPEG q0.82).
- `src/features/design/useDesigns.js`, `src/lib/useLocalStorageState.js` — `createDesign` now includes `backgroundImage` in its whitelist (was silently dropped on a new design's first Save — the root cause of an intermittent "background didn't persist" bug found by code review); `useLocalStorageState` now returns a 3rd `saveError` value, surfaced on the Designs list page if a browser storage-quota write ever fails.
- `src/App.jsx` — top-level page nav now confirms before navigating away from an unsaved design (was previously bypassable, since only the in-editor "Back" button had a dirty-check).

## ✅ Things I've Changed
- Added the 4 requested design-editor features (see Current State) plus PolyDock/ShoreMaster gangway and accessory catalog research (`src/data/partsCatalog.js`), all with the same real-source-or-flagged-`isVerified:false` discipline as the rest of the catalog.
- Ran an 8-angle parallel code review on the 4 features and fixed everything that survived verification:
  - **Real bug, most significant:** `createDesign()` never included `backgroundImage` in its field whitelist (only `saveDesign()` did, via spread) — a background image survived edits to an *existing* design but was silently dropped forever on a *brand-new* design's first Save. This was the root cause of an intermittent, hard-to-reproduce "background image disappeared" report. Fixed.
  - **Real bug:** the top-level Inventory/Designs page-nav buttons could discard an unsaved design with zero warning — only the in-editor "← Back" button had a dirty-check. Fixed by threading `onDirtyChange` from `DesignEditor` up through `DesignsPage` to `App.jsx`, which now confirms before switching away from Designs while dirty.
  - **Real bug:** rapid/duplicate background-file-input events (or re-selecting the same file) could race or silently no-op. Fixed with a generation-counter ref guard plus resetting `input.value` after read.
  - **Real bug:** replacing an existing background image reset its scale/opacity/position back to defaults instead of preserving them. Fixed.
  - **Real bug:** the background `<image>` had no `pointer-events: none`, so a click landing on the visible photo would defeat the click-to-deselect handler (event.target was the image, not the SVG background). Fixed.
  - **Real bug:** two independent drag state machines (module drag, background drag) had no mutual-exclusion — a pointer sequence that started one while the other was mid-drag could corrupt both. Fixed with targeted early-return guards in each pointer-down handler.
  - **Real, currently-reachable performance issue:** `validItems`/`footprintItems`/`markerItems` were recomputed unmemoized on every render, which silently defeated `computeSeams`' own `useMemo` (new array references every time) — meaning seam classification re-ran on every background-drag pointermove too. Fixed by memoizing all three.
  - **Real duplication:** `Palette.jsx` (née `ModulePalette.jsx`) reimplemented the same tab markup already factored into `CategoryTabs.jsx` for the inventory page. Fixed by reusing it directly.
  - **Robustness gap:** no graceful handling if a localStorage write ever fails (e.g. quota exceeded from too many large background images) — silent failure. Fixed: `useLocalStorageState` now surfaces a `saveError`, shown on the Designs list page (not the editor, since Save navigates away before an async write could be reflected there).
  - Minor: fixed a stale doc reference to the deleted `ModulePalette.jsx` in `feature-grid-canvas.md`.
- Verified all 4 features end-to-end via a real production-config build (not just dev server): drag-then-reselect-then-rotate (confirmed footprint dimensions swap correctly after a drag+reselect, the originally-reported bug), gangway placement with correct true-scale footprint and dashed-purple attachment-seam styling, accessory placement as untethered markers, background upload/position/scale/opacity, and full save→reload persistence of all of the above (including the background image) with zero console errors.
